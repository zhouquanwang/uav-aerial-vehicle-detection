# 航拍视频检测程序逻辑说明

本文档描述 `uav-aerial-vehicle-detection` 项目中视频推理检测程序的完整逻辑，便于阅读代码与调整配置。

**主入口**：`predict.py` 或 `python -m src.predict_video`  
**核心脚本**：`src/predict_video.py`  
**默认配置**：`configs/inference.yaml`

---

## 一、整体流程

```
启动 predict_video
    ↓
依赖检查（_ensure_runtime，必要时切换 .venv 重跑）
    ↓
main()：读取 YAML + 解析命令行参数
    ↓
选择模型 / 输入输出路径 / 推理设备
    ↓
run_inference()：加载模型 → 打开视频 → 逐帧循环
    ↓
每一帧：跳帧判断 → 检测+标注 或 原帧直出 → 写入输出视频
    ↓
释放资源，输出统计信息
```

**要点**：

- 输出视频的**帧数、分辨率、帧率**与输入一致（受 `max_frames` 限制时除外）。
- **跳帧**只减少「跑模型的次数」，不减少「写入的帧数」；跳过帧不画检测框。

---

## 二、启动阶段

| 步骤 | 模块 / 函数 | 作用 |
|------|-------------|------|
| 1 | `_ensure_runtime()` | 检查 cv2、sahi、supervision 等依赖；若当前 Python 缺包，自动用项目 `.venv` 重新启动 |
| 2 | `load_config()` | 读取 `configs/inference.yaml` |
| 3 | 模型选择 | 根据 `model_select`（1/2/3）或 `--model-select` 选择权重与类型 |
| 4 | `_resolve_path()` | 将配置中的相对路径解析为项目根目录下的绝对路径 |
| 5 | `resolve_device()` | `device: auto` → 有 GPU 用 `cuda:0`，否则 `cpu` |
| 6 | 输出路径 | 未指定 `--output` 时，默认 `outputs/predict_{模型名}.mp4` |

### 可选模型（`MODEL_OPTIONS`）

| model_select | 名称 | SAHI model_type | 默认权重路径 |
|--------------|------|-----------------|--------------|
| 1 | WALDO-YOLOv8m | yolov8 | `models/WALDO30_yolov8m_640x640 (1).pt` |
| 2 | RT-DETR | rtdetr | `models/4epoch-rtdetr-best.pt` |
| 3 | YOLOv8l | yolov8 | `models/4epoch-yolov8l-best.pt` |

命令行 `--model` 可指定任意 `.pt` 路径，此时默认按 yolov8 类型加载。

## 三、核心循环 `run_inference()`

### 3.1 初始化（仅执行一次）

1. 校验模型文件、输入视频是否存在。
2. 通过 SAHI 加载检测模型：
   ```text
   AutoDetectionModel.from_pretrained(
       model_type, model_path, confidence_threshold, device
   )
   ```
3. `cv2.VideoCapture` 打开输入；`cv2.VideoWriter` 创建输出（宽高、fps 与输入一致）。
4. 创建标注器：
   - `BoxCornerAnnotator`：绘制检测框；
   - `RichLabelAnnotator` + 系统中文字体（`zh_draw.py`）：绘制中文类别与置信度。

### 3.2 逐帧处理逻辑

对每一帧（`frame_index` 从 0 递增）：

```text
读取一帧 frame
    ↓
是否达到 max_frames？ → 是则结束循环
    ↓
should_run_detection(frame_index, detect_skip_interval)？
    ├─ 否（跳帧）→ output_frame = 原帧副本，不调用模型
    └─ 是（检测）→ output_frame = annotate_frame(...)
    ↓
out.write(output_frame)    # 每一帧都写入输出
frame_count += 1
```

### 3.3 跳帧规则 `detect_skip_interval`

| 配置值 | 含义 | 做检测的帧序号（从 0 起） |
|--------|------|---------------------------|
| 0 | 不跳帧 | 0, 1, 2, 3, … |
| 1 | 检测 1 帧、跳过 1 帧 | 0, 2, 4, 6, … |
| N (N≥1) | 检测 1 帧、跳过 N 帧 | 0, N+1, 2(N+1), … |

**判定公式**：`frame_index % (detect_skip_interval + 1) == 0`

跳过的帧：不运行 SAHI/YOLO，输出画面与输入该帧相同（无检测框、无中文标签）。

## 四、单帧检测 `annotate_frame()`

算力最集中的步骤，流程如下：

```text
当前帧（BGR numpy 数组）
    ↓
SAHI：get_sliced_prediction()
    ├─ 按 slice_height/width（默认 640）切分整帧
    ├─ overlap 默认 0.1，减少目标被切在边界上的漏检
    ├─ 每片调用 YOLO/RT-DETR 推理
    └─ 合并切片结果到整图坐标（含 NMS 等后处理）
    ↓
按 target_classes 过滤（只保留配置中的类别 ID）
    ↓
有检测框？
    ├─ 无 → 返回原帧副本
    └─ 有 → 组装 sv.Detections
            → 中文标签（class_labels_zh + 置信度）
            → BoxCornerAnnotator 画框
            → RichLabelAnnotator 画字
            → 可选：右上角模型名称水印
    ↓
返回标注后的帧
```

### 4.1 为何使用 SAHI 切片

航拍视频分辨率高，车辆在整帧中往往很小。将每帧切成 640×640 重叠小块分别检测，再合并结果，有利于小目标召回。终端中的 `Performing prediction on N slices` 即每帧切片数量（与分辨率、重叠比例有关）。

### 4.2 类别过滤（推理之后）

WALDO 预训练模型共 12 类。程序在 SAHI 返回结果后，仅保留 `target_classes` 中的 ID。

当前默认检测 8 类交通相关目标：

| WALDO ID | 模型原名 | 视频中文标注 |
|----------|----------|--------------|
| 0 | LightVehicle | 小汽车 |
| 1 | Person | 行人 |
| 5 | Bike | 非机动车 |
| 6 | Container | 集装箱车 |
| 7 | Truck | 货车 |
| 8 | Gastank | 罐车 |
| 9 | Digger | 工程车 |
| 11 | Bus | 大客车 |

未在 `target_classes` 中的类别（如建筑、船等）会被丢弃，不绘制。

**说明**：YOLO 一次前向仍会输出全部类别；过滤发生在后处理阶段，**不会**因只显示 8 类而成倍增加推理时间。

### 4.3 中文标注

- OpenCV 默认字体不支持中文，`LabelAnnotator` 会显示方框或乱码。
- 程序使用 `RichLabelAnnotator` + TrueType 字体（Windows 优先 `msyh.ttc` 微软雅黑）。
- 标签格式示例：`小汽车 0.87`。
- 可在配置中设置 `label_font_size`、`font_path`。

### 4.4 模型水印

检测帧可在右上角叠加 `[Model: WALDO-YOLOv8m]` 等文字（`add_model_watermark=True`），便于区分不同模型输出。

## 五、配置与命令行

**优先级**：命令行参数 > `inference.yaml` > 代码内默认值

| 参数 | 说明 |
|------|------|
| `model_select` | 1 / 2 / 3，选择预置模型 |
| `models` | 各模型权重相对路径 |
| `input_video` | 输入 MP4 |
| `output_video` | 输出路径（可被自动生成规则覆盖） |
| `device` | `auto` / `cpu` / `cuda:0` |
| `confidence_threshold` | 置信度阈值，默认 0.2 |
| `target_classes` | 保留的 WALDO 类别 ID 列表 |
| `class_labels_zh` | ID → 中文显示名 |
| `slice_height` / `slice_width` | SAHI 切片尺寸，默认 640 |
| `overlap_height_ratio` / `overlap_width_ratio` | 切片重叠比例，默认 0.1 |
| `detect_skip_interval` | 跳帧间隔，0 为不跳帧 |
| `max_frames` | 最多写入帧数；`null` 或删除表示全片 |
| `label_font_size` | 中文标签字号 |

### 常用命令

```powershell
cd f:\cursor\uav-aerial-vehicle-detection
.\.venv\Scripts\Activate.ps1

# 使用配置文件默认参数
python -m src.predict_video

# 指定模型与短跑帧数
python -m src.predict_video --model-select 1 --max-frames 10

# 指定输入输出
python -m src.predict_video --input data\your.mp4 --output outputs\out.mp4

# 关闭跳帧、全片推理（需在 yaml 中设 max_frames: null）
python -m src.predict_video --detect-skip-interval 0
```

|------|------|
| `predict.py` | 启动器，优先使用项目 `.venv` |

## 七、运行示例（结合典型配置）

假设配置为：

- `model_select: 1`（WALDO）
- `detect_skip_interval: 5`
- `max_frames: 500`
- `target_classes: [0, 1, 5, 6, 7, 8, 9, 11]`

则一次运行大致为：

1. 读取 `data/input.mp4`，最多处理 **500 帧** 后停止。
2. 第 0、6、12、18… 帧执行检测（每 6 帧检测 1 次）；中间帧原样写入、不标注。
3. 每个检测帧：切片推理 → 过滤 8 类 → 中文画框 → 写入 `outputs/predict_WALDO-YOLOv8m.mp4`。
4. 终端进度示例：`Frame 100 [detect] | detected 17`。

## 八、性能与常见问题

| 现象 | 原因 | 建议 |
|------|------|------|
| 极慢 | CPU 推理 + 每帧多切片 | 安装 GPU 版 PyTorch，`device: auto` |
| 大量 `Performing prediction on N slices` | SAHI 默认 `verbose=1` | 正常；可在代码中对 `get_sliced_prediction` 传 `verbose=0` |
| 中文方框 | 未走 RichLabel 或字体缺失 | 配置 `font_path` 指向 `.ttc` / `.ttf` |
| `KeyboardInterrupt` | 用户 Ctrl+C 中断 | 非程序错误；需重新跑完以得到完整 MP4 |
| 跳帧后画面「闪一下」无框 | 跳过帧未做检测 | 预期行为；可减小 `detect_skip_interval` |
